# Security Policy

Security matters in operational engineering. Please report vulnerabilities responsibly and avoid public disclosure until a fix is available.

## Reporting a Vulnerability

- Do not open a public issue for security findings.
- If available, use a GitHub Security Advisory for private reporting.
- Otherwise, contact the maintainers via their GitHub profiles.
- Provide a clear description, impact assessment, and reproduction steps.

## Response Expectations

- An initial acknowledgment is targeted within 7 days.
- Fix timelines depend on severity and scope.
- Coordinated disclosure is preferred once a fix is available.

## Scope

This policy applies to repository content, scripts, and examples. External services and third-party platforms are out of scope.

## Operational Security Principles

- Avoid scanning systems you do not own or control.
- Do not exploit vulnerabilities beyond what is needed to demonstrate impact.
- Redact secrets and sensitive data in all reports.
